Como crear tu primer Portfolio con control de versiones en Github desde el terminal de tu sistema operativo

  1. Inicia sesión en Github.com o crea una cuenta en caso de que no tengas una   
  2. Crea un nuevo repositorio, dale un nombre tipo usuario.github.io y copia la dirección URL del HTTPS     
  3. Abre el terminal de Git Bash y ve a la carpeta donde quieres manejar tu repo. En mi caso documentos/MITxPRO   
  4. Utiliza el comando git clone y pega la URL que copiaste de tu repo en github. La línea de código debería verse algo como "git clone ttps://github.com/username.github.io .git"      
  5. Ingresa a la carpeta del repo en tu terminal usando el comando cd. La línea de código debería verse algo como “cd username.github.io”   
  6. Añade un archivo con el comando touch. Por ejemplo: touch index.html   
  7. Usando el explorador de archivos de tu sistema operativo abre la carpeta del repositorio y coloca una imagen para llamarla desde el index.html   
  8. Abre el archivo index.html con VS code y coloca una estructura básica con un título y añade la imagen que copiaste usando la etiqueta img. Una vez creado guarda el archivo index.html   
  9. Ahora vamos a añadir esos 2 archivos desde tu carpeta local a tu carpeta en github. Para ello podemos usar el comando “git add filename.ext” para cada archivo, por ejemplo: “git add index.html” “git add PerfilDani.jpg” o también puedes añadirlos todos al mismo tiempo con el comando “git add .” Una vez hecho esto puedes revisar el estado del git con el comando “git status”   
  10. Ahora confirma los cambios usando el comando “git commit -m “mensaje”   
  11. Por último carga tus archivos al repositorio usando el comando “git push origin branchname” en mi caso el código se verá así “git push origin master”      
  12. Revisa tu repositorio en github.com y confirma que se hayan subido los archivos   
  13. Revisa el resultado final ingresando la dirección de tu repo en el explorador, en mi caso, soydanielromero.github.io
